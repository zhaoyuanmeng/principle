#### Git提交规范

1、feat:      新功能
2、fix:       修改bug
3、docs:      修补文档
4、style:     格式化代码结构（不影响代码运行的变动）
5、refactor:  重构（比如重命名变量）
6、chore:     构建过程中或者辅助工具的变动（不影响代码的运行）
